# PRD：订阅制亚马逊墨西哥热榜数据平台

> 状态：草稿 / 最后更新：2026-06-27
> 背景：现有爬虫已能抓取 Amazon MX 畅销榜数据（宠物狗类目），本产品在此基础上扩展为多分类、订阅制、可交付的 SaaS 工具。

---

## 一、产品定位

面向**跨境电商卖家**（主要是国内做墨西哥站的选品人），提供亚马逊墨西哥站畅销榜的**实时数据 + 趋势对比**，以订阅分类的形式收费，通过网页查看、Excel 导出、邮件推送三种方式交付数据。

**差异化核心**：不卖静态数据，卖「变化」——谁新上榜、谁月销暴涨、谁调了价格。

---

## 二、目标用户

- 正在做或准备做亚马逊墨西哥站的跨境卖家
- 需要定期做选品调研，但没有时间或能力自己抓数据
- 主要通过小红书、闲鱼、跨境电商社群触达

---

## 三、核心功能

### 3.1 数据层（已有基础）

| 功能 | 状态 | 说明 |
|------|------|------|
| 抓取畅销榜 Top 100 | ✅ 已有 | 现有爬虫，默认宠物狗分类 |
| 多分类支持 | ✅ 已有分类链接 | 25个一级 + 365个二级分类链接已抓取 |
| 历史快照存储 | ✅ 已有文件 | 当前为 CSV/JSON 文件，需迁移至数据库 |
| History 对比分析 | ❌ 待开发 | 见 3.2 |

### 3.2 对比分析指标

以 **ASIN** 为唯一键，对比相邻两次快照：

| 指标 | 触发条件 | 注意事项 |
|------|---------|---------|
| 新上榜 | 本期有、上期无此 ASIN | — |
| 新下榜 | 上期有、本期无此 ASIN | — |
| 排名变化 | 排名变动 ≥ 5 位 | 阈值待定 |
| 月销异动 | 变化幅度 ≥ 20% | ⚠️ 月销来源需一致（官方数据 vs 推算数据不可混用对比） |
| 价格调整 | 变化幅度 ≥ 5% | — |
| 新增变体 | 本期变体数 > 上期变体数 | — |
| 评分下滑 | 评分下降 ≥ 0.1 | — |

### 3.3 用户系统

- 注册 / 登录（邮箱 + 密码）
- 邮箱验证
- 订阅分类管理（查看已订阅、新增、取消）
- 有效期管理

### 3.4 数据展示（网页）

- 未登录：展示宠物狗分类前 10 名（引流钩子）
- 免费用户：1个分类完整 Top 100
- 付费用户：按订阅分类解锁，支持切换查看

页面功能：
- 表格展示（排名、ASIN、品牌、价格、月销、变体数、对比标签）
- 筛选 / 排序
- 对比标签高亮（新上榜🆕、价格下调📉等）

### 3.5 数据导出

- Excel 导出（当前分类当前快照 + 对比列）
- 发送到邮箱（手动触发，发送当前视图数据）
- ⚠️ **待定**：是否支持定时自动邮件推送（每周/每天），推送内容格式未定

## 四、技术方案

### 4.1 技术栈

| 层 | 选型 | 理由 |
|----|------|------|
| 后端 | Flask (Python) | 与现有爬虫同语言，改动最小 |
| 前端 | 现有 index.html 扩展 | 已有表格和图表基础 |
| 数据库 | SQLite → PostgreSQL | MVP 用 SQLite，规模化后迁移 |
| Excel 生成 | openpyxl | 项目已依赖 |
| 邮件服务 | Resend 或 SendGrid | 免费额度各 100 封/天，够 MVP |
| 定时任务 | 现有 plist 脚本扩展 | macOS 本地跑，后期迁服务器 |

### 4.2 数据存储迁移

当前：爬虫结果存为 CSV/JSON 文件
目标：爬虫结果写入 SQLite，Flask 查数据库

迁移方式：改造爬虫脚本，在保存文件的同时写入数据库（兼容过渡）

### 4.3 部署

⚠️ **待定**：
- 当前在本地 Mac 运行，公开网页需要服务器或云托管
- 域名、SSL 未决定
- 备选：Railway / Render（免费套餐） / 阿里云轻量服务器

- 微信 / 支付宝收款码
- 用户付款截图发给你，手动在数据库更新有效期

**Phase 2（自动化）**：
- 接入虎皮椒或 PayJS
- ⚠️ **待确认**：两个平台均要求企业支付宝或营业执照，纯个人账号能否过审需验证

## 八、未决问题清单

1. **月销对比数据源一致性**：官方数据和推算数据混用会导致对比失真，需在对比逻辑中过滤或标注
2. **更新频率承诺**：给用户承诺的数据更新频率是每天还是每周？影响爬虫调度和服务器成本
3. **邮件推送格式**：自动周报的内容和样式未定
4. **服务器 / 域名**：公开上线需要，选型和费用未定
5. **PayJS / 虎皮椒个人申请可行性**：需实测是否能过审
6. **闲鱼运营策略**：发布形式、定价、商品描述未定
7. **推广社群渠道**：具体社群未选定
8. **冷启动策略**：是否先免费放开吸引种子用户，再转付费
